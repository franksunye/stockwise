# StockWise UX 概念设计方案：TikTok + Weather App 融合体验

> **设计理念：交易者的“沉浸式”抬头显示器 (HUD)**
>
> 本方案旨在将信息消费型应用的沉浸式交互引入金融决策工具。通过“横向扫视多只标的”与“纵向追溯历史逻辑”的组合手势，为投资者提供极致流畅的决策心流，消除信息噪音，强化交易纪律。

---

## 1. 设计愿景：从“列表”到“空间”

传统的股票 App 采用层级导航（列表 -> 详情 ->深度分析），这增加了用户的认知负担。
StockWise X 的新 UX 方案通过一个**二维空间模型**来重构：

- **X 轴 (苹果天气模式)**：并行观察。左右滑动切换不同的股票，就像查看不同城市的天气。
- **Y 轴 (抖音模式)**：深度溯源。上划追溯当前股票的历史跟踪和决策依据。
- **Z 轴 (博主页模式)**：点击图标进入“股票档案”，查看长期的复盘统计和小方块汇总。

## 2. 核心架构与操作指南

### 2.1 智能决策主场 (Main HUD Context)
这是用户在交易日内停留 90% 时间的界面。

- **横向左右滑 (Horizontal Swipe)**：
  - **交互逻辑**：在用户关注池（如 5-10 只股票）中无缝切换。
  - **视觉引导**：底部配有 iOS 风格的页面指示器（小圆点），提示当前位置。
  - **价值**：盘中快速“扫射”所有关注资产的即时信号，无需进出目录。

- **纵向上下滑 (Vertical Timeline)**：
  - **交互逻辑**：向上划动显示 T-1, T-2... 的历史跟踪内容。
  - **物理隐喻**：每一天的预测是一张“便签卡片”，上划动作就像在翻阅案头的便签本。
  - **价值**：当今日信号发生冲突时，通过快速回溯历史逻辑，找回交易初衷。

### 2.2 股票档案与复盘 (Stock Profile)
通过点击顶部的股票 ICON/名称进入。
- **视觉呈现**：模仿抖音博主主页，采用 3xN 的小方块布局。
- **方块逻辑**：每格代表一天的复盘结果（🟢 命中预期 / 🔴 产生偏离）。
- **价值**：建立用户对 AI 信号和自身纪律的长期信任。

## 3. 导航与布局细节 (Navigation & Layout)

### 3.1 极简布局架构 (HUD Layout)
- **顶部 (Header)**：
  - **左侧**：股票名称与 ICON。**[核心入口]** 点击此点击即可进入“股票档案/复盘矩阵”，是追溯长周期胜率的唯一入口。
  - **右侧**：全局菜单，处理不常变动的规则设定。
- **中间核心 (The Core)**：全屏沉浸区，负责展示红绿灯信号与 AI 摘要。
- **底部 (The Footer)**：
  - **左下角**：关注池入口。管理空间维度的资产增减。
  - **中间**：页面位置指示器。
  - **右下角**：**保持空灵**。或放置极其隐蔽的“数据同步时间”标识。

### 3.2 手势冲突管理策略 (Conflict Management)
为了防止左右滑与上下滑的误触：
1. **边缘防护**：左右滑动仅在屏幕左右 15% 的边缘或特定中性区触发。
2. **滚动吸附 (Scroll Snap)**：使用强制吸附，确保每一屏信息展示完整，不会停留在半截。
3. **视觉反馈**：滑动时背景辉光随手势偏移，增强物理感。

---

## 4. 关键体验增强 (UX Boosters)

- **触感反馈 (Tactile Feedback)**：
  - 滑动到边界时添加 iOS 风格的回弹效果。
  - 核心信号转变（如从🟡转🔴）时触发微弱震动。
- **预览指示器**：在卡片底部边缘隐约显示下一页/上一页的颜色或名称缩写，引导用户滑动。
- **快捷跳转**：长按底部复盘入口，弹出“日期快速跳板”，而非逐页翻阅。
- **空状态引导**：当关注池为空时，采用类似天气 App 的“大搜索框”设计，引导用户添加第一只标的。

---

## 5. 技术实现可行性建议

| 模块         | 推荐技术栈                                         |
| :----------- | :------------------------------------------------- |
| **手势识别** | `Framer Motion` (处理拖拽动画) 或 `use-gesture`    |
| **层级容器** | CSS `scroll-snap-type: both mandatory`             |
| **历史加载** | `React Query` 的无限滚屏 (Infinite Query) 机制     |
| **数据同步** | SWR / Optimistic UI (确保滑动时的实时数据无感更新) |

---

## 6. 与产品定位的深度契合

本方案完美落地了 **Product_Spec.md** 中定义的“理性锚点”愿景：
1. **专注力**：通过全屏沉浸感，一次只让用户看一只股票，杜绝分心。
2. **确定性**：统一的交互模式让操作变成“肌肉记忆”，减少交易时的情绪波动。
3. **历史可复制性**：上划追溯历史的功能，强制用户反思过去的判断，培养长期眼光。

---

> **结论**：StockWise X 不仅仅是一个数据工具，它通过“抖音化”的交互，将原本沉长的交易复盘和监控周期缩减到了“指尖滑动”之间，是真正意义上的交易员 HUD。
